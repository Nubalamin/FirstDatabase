The materialized view stores the results of a query physically, so it must be updated whenever the Part table changes.
That is, if a new part has partcost that is greater than 1000, then its partNo is added to the view. Furthermore, if an expensive part is deleted,then it is necessary to check whether another contract for the same part still costs mor  than 1000 before removing it from the view.

The view can be maintained without accessing the part table during inserts or updates that make a part expensive because the new data is enough.
